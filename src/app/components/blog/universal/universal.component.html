Angular Universal

  1. It is node based render engine to use ssr or prerender

  2. We use ssr for busting performance, seo optimization, for social media crawlers

  3. Prerender - build time, ssr - run time

  4. We cannot prerender dynamic routes. One of the downsides to static rendering is that individual HTML files must be generated for every possible URL. Prerendering improves the First Paint or First Contentful Paint of a Single Page Application that must be booted on the client in order for pages to be truly interactive.

  5. Title inside head tag is very important for seo and within angular 14 we can add title to each route

  6. Shall architecture - we prerender or use ssr for small spinner or skeleton to show the user out page is working till main js bundle is loaded and frond end is in charge of the app

  7. SSR does not improve TTI time (time to be interactive), but improve FCP and FP

  8. Use Angular features (title, a11y sdk)

  9. Test your site with keyboard, lighthouse, screen reader (NVDA, JAWS, Voice Over), etc
